# モデル選定とストリーミング出力の重要性（運用メモ）

## 1. 選定モデル

- **選定モデル**: `GLM-4.7`

## 2. 選定の背景と理由

本プロジェクトは、チャットボット（将来的にはAI VTuber等の「キャラクター」を介した対話）として、水温・pHなどの環境データを分かりやすく案内する仕組みへの展開も想定しています。  
このユースケースでは、**回答品質に加えて、体感待ち時間（TTFB: Time To First Byte）の短さがUXを大きく左右**します。

### 2.1 TTFB（最初の出力までの待ち時間）がUXに直結する

- **キャラクター対話は“間”が命**: 返答開始まで沈黙が続くと、人間側には「固まった」「聞こえていない」「不自然」という印象が残りやすい。
- **環境データ案内は“即応性”が価値**: 数値や状態の説明は、テンポよく返ってくるほど「安心感」と「信頼感」が上がる。

### 2.2 `GLM-4.7` はストリーミング出力に対応し、リアルタイム連携に向く

`GLM-4.7` はストリーミング出力に対応しており、**最初の数文字を受信した時点で即座に下流処理へ流し込める**点が運用上の強みです。

- **音声合成（例: VOICEVOX）**: 文章の到着を待たずに先頭から読み上げを開始できる
- **リップシンク/表情制御**: 発話開始タイミングを前倒しでき、自然な対話リズムを作れる
- **UI表示**: “書きながら話す”ように表示でき、ユーザーが「今生成中」を理解できる

結果として、**一括出力を待つ間の“不自然な沈黙”を減らし、自然な対話リズム**を実現できます。

### 2.3 `GLM-4-32B` 等と比較した場合の実運用メリット

本プロジェクトの運用では、マルチターン対話やロールプレイ（性格・一貫性の維持）を重視します。  
その上で `GLM-4.7` は、より大規模なモデル（例: `GLM-4-32B` 系）と比較して **軽量・高速**であり、**応答のテンポ**を維持しやすい利点があります。

- **目的適合**: チャット/キャラクター対話のテンポを優先しやすい
- **スループット**: 同時利用や連続会話の負荷に対して安定運用しやすい
- **体感品質**: “返答の開始が速い”ことが、会話体験の品質を底上げする

## 3. ストリーミングの重要性（運用観点のまとめ）

- **TTFBの改善**: 先頭チャンクが届いた時点で表示/音声を開始できる
- **体感レスポンスの改善**: 全文完成を待つ必要がなく、待機ストレスが減る
- **会話の自然さ**: “沈黙が続く”状況を減らし、対話のテンポが安定する
